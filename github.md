GitHub Features



1\. Repositories: Repositories are used to store project files and code.



2\. Branches: Branches allow developers to work on different features without affecting the main code.



3\. Pull Requests: Pull requests are used to propose and review changes before merging.



4\. Issues: Issues help track bugs, tasks, and feature requests.



5\. GitHub Actions: GitHub Actions automate workflows like testing and deployment.



6\. Releases: Releases are used to publish versions of the software.



7\. Wiki: Wiki provides a space for detailed project documentation.



8\. Projects: Projects help manage tasks using boards and cards.



9\. GitHub Pages: Host static websites for projects or portfolios.



10\. Gists: Share code snippets and scripts.
